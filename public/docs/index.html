Introduction

Base URL: http://localhost/v1/

This documentation aims to provide all the information you need to work with Babel.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile) to suit your programming needs.

Authenticating requests

This API is authenticated by sending an Authorization header with the value "Bearer ed43530dcaa9ec5d061dd55e1e64c0e9".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

Endpoints

Test RouterOS

This endpoint allows you to test the Mikrotik server connection with this API. It's a really useful endpoint and a start point to know if there is any problem.

I recommend you to use first, to debug any futher inconvenience. üòÅ
Example request:
curl --request GET \
    --get "http://localhost/v1/v1/test" \
    --header "Content-Type: application/xml" \
    --header "Accept: application/json" \
    --data "{
    \"ip\": \"192.168.1.1\"
}"

const url = new URL(
    "http://localhost/v1/v1/test"
);

const headers = {
    "Content-Type": "application/xml",
    "Accept": "application/json",
};

let body = {
    "ip": "192.168.1.1"
};

fetch(url, {
    method: "GET",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());
$client = new \GuzzleHttp\Client();
$response = $client->get(
    'http://localhost/v1/v1/test',
    [
        'headers' => [
            'Content-Type' => 'application/xml',
            'Accept' => 'application/json',
        ],
        'json' => [
            'ip' => '192.168.1.1',
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));

Enable Connections

This endpoint gets the ip/s in the address-list "clientes_cortados", then it puts it/them back in "clientes_activos" address-list

With this you are able to allow the "contracts" to have a valid connection to the internet
Example request:
curl --request PATCH \
    "http://localhost/v1/v1/connection/enableConnection" \
    --header "Content-Type: application/xml" \
    --header "Accept: application/json" \
    --data "{
    \"ip\": \"192.168.1.1\",
    \"clientes\": {
        \"cliente_ip\": \"1.1.1.1\"
    }
}"

const url = new URL(
    "http://localhost/v1/v1/connection/enableConnection"
);

const headers = {
    "Content-Type": "application/xml",
    "Accept": "application/json",
};

let body = {
    "ip": "192.168.1.1",
    "clientes": {
        "cliente_ip": "1.1.1.1"
    }
};

fetch(url, {
    method: "PATCH",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Disable Connections

This endpoint gets the ip/s in the address-list "clientes_activos", then it puts them in the "clientes_cortados" address-list

With this you are able to restrict the "contracts" to have a valid connection to the internet üòà
Example request:
curl --request PATCH \
    "http://localhost/v1/v1/connection/disableConnection" \
    --header "Content-Type: application/xml" \
    --header "Accept: application/json" \
    --data "{
    \"ip\": \"192.168.1.1\",
    \"clientes\": {
        \"cliente_ip\": \"1.1.1.1\"
    }
}"

const url = new URL(
    "http://localhost/v1/v1/connection/disableConnection"
);

const headers = {
    "Content-Type": "application/xml",
    "Accept": "application/json",
};

let body = {
    "ip": "192.168.1.1",
    "clientes": {
        "cliente_ip": "1.1.1.1"
    }
};

fetch(url, {
    method: "PATCH",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());
$client = new \GuzzleHttp\Client();
$response = $client->patch(
    'http://localhost/v1/v1/connection/disableConnection',
    [
        'headers' => [
            'Content-Type' => 'application/xml',
            'Accept' => 'application/json',
        ],
        'json' => [
            'ip' => '192.168.1.1',
            'clientes' => [
                'cliente_ip' => '1.1.1.1',
            ],
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));

Get Data Mikrotik

This endpoint gets info from the router (clients connected, active clients, clipped, etc)

VERY USEFUL endpoint
Example request:
curl --request GET \
    --get "http://localhost/v1/v1/router/getDataMikrotik" \
    --header "Content-Type: application/xml" \
    --header "Accept: application/json" \
    --data "{
    \"ip\": \"192.168.1.1\",
    \"clientes\": {
        \"cliente_ip\": \"1.1.1.1\"
    }
}"

const url = new URL(
    "http://localhost/v1/v1/router/getDataMikrotik"
